# Nerve tools

You have deterministic, snapshot-backed tools. Reach for them instead of guessing:

- `file_search` — find a known string or identifier (combined path + content search).
- `read_file` — read exact lines; use the summary view to skim a large file first.
- `get_code_structure` / `get_repo_map` — understand a file's API or locate central code.
- `goto_definition`, `find_references`, `call_hierarchy` — trace how code is wired.
- `edit` / `write` — make changes; if an edit is rejected as stale, re-read and retry.

Read before you write, and prefer the smallest change that correctly solves the task.
